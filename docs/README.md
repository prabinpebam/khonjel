# Khonjel — Documentation

This folder holds Khonjel's **living** specifications and policies, plus an **archive** of
completed, point-in-time material.

## Structure

```
docs/
├── product-spec/     Living product specification (vision · product · IA · UX/UI · architecture)
├── security/         Living security policies (threat model, data inventory, Electron policy, IR)
├── frameworks/       Methodology — Eval-Driven Development (EDD)
└── archive/          Point-in-time records (completed delivery plans, dated audits, source analysis)
```

## Where to start

- **Building or resuming?** Read
  [`product-spec/00-execution-playbook.md`](product-spec/00-execution-playbook.md) — how we code
  and execute this project (the doc to revisit when context is lost).
- **The product spec** — [`product-spec/README.md`](product-spec/README.md).
- **Security policies** — [`security/`](security/): threat model, data inventory, Electron policy,
  incident response, and the new-integration checklist.
- **EDD methodology** — [`frameworks/eval-driven-development/`](frameworks/eval-driven-development/README.md).

## Archive

[`archive/`](archive/) holds **completed, point-in-time** documents kept for history — the delivered
backend / GPU-acceleration / Parakeet plans, dated security audits, and the source-app reference
analysis. See [`archive/README.md`](archive/README.md). These are historical; prefer the living spec
above for current truth.
